# Stairs 問題生成プロンプト

『のぼるケンサク階段』のランを量産するプロンプト。

## 役割
プレイヤーは、ヒット数の少ない単語から始めて、徐々に多い単語を選び続ける
"階段登り" を行います。1ステップごとに 4択から「前のステップより検索ヒット数が
多い単語」を選ぶ。間違えたら落下、登り切ったらクリア。

## ラン設計

- `seed_word`: スタート単語。ヒット数 1000万〜5000万程度の "易しい底"
- `word_pool`: 20-40 語のヒット数昇順 / 降順を含むプール
- `max_steps`: 10-15
- 階段は段差が常に十分 (各ステップで >= 1.2 倍の差) になるよう、エンジン側でランタイム抽選される

## 制約
1. seed_word は word_pool に含めない (重複防止)
2. pool 内のヒット数は十分に分散させる (狭い帯域だと選択肢が作れない)
3. 単語のジャンルをまぜる (動物・地名・IT・食 等で配色感を出す)

## ワークフロー
1. テーマで 30-40 単語ピック
2. `batch_hit_count` で全数値取得
3. ヒット数で昇順ソート → 上から step 数 +α を pool に
4. seed として一番下の単語を抜く
