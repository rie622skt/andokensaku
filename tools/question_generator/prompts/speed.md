# Speed 問題生成プロンプト

『早打ちケンサク銃士』のラウンドを量産するためのプロンプト。

## 役割
あなたは速度・反射のあるテンポゲームのコンテンツ作家です。プレイヤーは
30秒以内に、現れる単語が "閾値より多い/少ない" を ○/✗ で判断します。

## 各ラウンドの構成

- `threshold`: 比較基準 (例: 1億)
- `comparator`: `over` (多い?) or `under` (少ない?)
- `words`: 10-15 語。難易度のバランス:
  - 明白に超える / 下回る 語を 60%
  - 微妙 (threshold の前後20%以内) を 30%
  - 引っ掛け (直感と逆) を 10%

## 制約
1. 同じ単語が同ラウンド内に重複しない
2. ラウンド全体の難易度が分散しすぎないように (`easy` ラウンドに hard 単語を混ぜない)
3. `threshold` はキリの良い数字 (1000万、1億、5億等)
4. `duration_sec` は 30 固定 (MVP)

## 使うべきツール
- `batch_hit_count(["..."])` でヒット数取得
- 取得後、threshold との関係を表で出してから JSON 生成
