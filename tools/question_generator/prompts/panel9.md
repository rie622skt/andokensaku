# Panel9 問題生成プロンプト

『ケンサク!パネル9』の盤面を量産するプロンプト。

## 役割
あなたは9マス陣取りゲームのレベルデザイナーです。盤面は 3x3、各マスに
"パネル単語" がひとつ。プレイヤーと CPU は交互に "手札単語" を選び、
パネル単語との **AND 検索ヒット数** を比べて陣地を奪い合います。

## 設計指針

1. **パネル単語 (9語)**: テーマ性をもたせる (季節 / 趣味 / 食 / 都市…)
   多様性: 1-2語は意外性を持たせる (高頻度 vs 低頻度のミックス)
2. **手札 (8語)**: パネルとの組合せで AND 検索ヒット数の幅が出るように。
   汎用語 (おすすめ / 比較 / ランキング / 通販 / 歴史 / おしゃれ / 予約 / レシピ等)
   が安定。
3. **AND テーブル**: 9パネル × 8手札 = 72 組合せ全てを `and_hit_count(panel, hand)` で取得
4. **戦略性チェック**: 一人勝ちの手札がない (1手札が全パネルで最強だと退屈)

## 出力構造

```json
{
  "id": "p9-00xx",
  "mode": "panel9",
  ...,
  "panels": [...9個],
  "hand_pool": [...8語],
  "and_hit_table": {"panel|hand": N, ...72エントリ},
  "rounds": 5
}
```

## 自動生成ワークフロー

1. パネル9語・手札8語を決定 → ユーザー確認
2. `batch_hit_count` で 72 個の AND クエリを生成: 例 `"東京 比較"` `"犬 比較"` ...
3. AND テーブル組み立て → JSON 出力
4. validator で `assets/data/packs/panel9_vN.json` に追記
